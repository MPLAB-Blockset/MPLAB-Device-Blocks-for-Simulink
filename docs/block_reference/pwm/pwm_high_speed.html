MCHP_PWM_HighSpeed - High-Speed PWM Block

MCHP_PWM_HighSpeed Block Icon

Block Description

The MCHP_PWM_HighSpeed block provides advanced high-speed PWM peripheral control for modern dsPIC families (dsPIC33C, dsPIC33CH, dsPIC33CK). This block offers significantly more flexibility than the standard PWM block, including independent time bases, multiple output modes, and enhanced configuration options.

This block is designed for demanding motor control and power conversion applications requiring precise timing, independent channel control, and high switching frequencies.

Supported Device Families

Family Masksets Key Features
dsPIC33F (selected) DOS_00958_pwm_TLAH
DOS_00958_pwm_TLAY		 Basic high-speed PWM, Auxiliary clock support
dsPIC33EP (GS) DOS_00958_pwm_TLAH
DOS_00958_pwm_TLAY		 Enhanced features, APLL clock, up to 12 channels
dsPIC33C/CH/CK Various (WAC, WBH masksets) Full-featured high-speed PWM, independent time bases
Note: For dsPIC30F/33F with standard PWM, use MCHP_PWM. For dsPIC33A, use MCHP_PWM_HS_FEP (Fine Edge Positioning).

Key Features

  • Up to 12 PWM channels (device dependent)
  • Four output modes: Complementary, Redundant, Push-Pull, Independent
  • Independent time base per PWM pair or master time base
  • Edge-aligned or center-aligned operation
  • Independent or master duty cycle control
  • Double update mode for asymmetric center-aligned PWM
  • Auxiliary clock support (APLL) for higher resolution
  • Immediate or synchronized updates
  • Dead-time insertion (configurable per channel)
  • Multiple trigger outputs for ADC/DAC synchronization

Output Modes

Mode Description Use Case
Complementary PWMxH and PWMxL are complementary with dead-time insertion Standard motor drive, half-bridge converters
Redundant PWMxH and PWMxL output identical signals Parallel drive, increased current capability
Push-Pull PWMxH and PWMxL alternate (no dead-time) Push-pull converters, full-bridge DC-DC
Independent PWMxH and PWMxL operate independently Dual independent loads, asymmetric control

Clock Configuration

Auxiliary Clock (APLL) - dsPIC33EP GS Only

For devices with DOS_00958_pwm_TLAH or DOS_00958_pwm_TLAY:

FOSC = APLL_frequency * 8;  % Auxiliary PLL clock
% Provides higher resolution and flexibility
% Typical APLL: 120 MHz → FOSC = 960 MHz for PWM

Standard Clock

FOSC = FCY * 2;  % Standard instruction clock
% FCY from Master block configuration

Implementation Details

Register Configuration

The PWM HighSpeed block configures numerous registers depending on mode:

Register Type Examples Purpose
Control IOCONx, FCLCONx, PDCx, PHASEx, DTRx, ALTDTRx PWM mode, output control, duty cycle, phase, dead-time
Period PTERx, MSTRx Time base period, master period selection
Triggers TRIGx, TRGCON ADC/DAC synchronization triggers
Faults FCLCON, LEBCON Fault input configuration, leading-edge blanking

Workspace Variables

The block creates workspace variables for each enabled PWM:

% For independent time base (ITB = 1):
PWM1_Tcy = PTPER1_value;  % Timer counts for PWM1
PWM2_Tcy = PTPER2_value;  % Timer counts for PWM2

% For master time base (ITB = 0):
PWMx_Tcy = MSTR_value;    % Shared timer counts

% Maximum duty cycle:
PWM1_max = (PWM1_Tcy * 2 - 1);

Usage Examples

Example 1: 3-Phase Motor Control (Complementary Mode)

% Configuration:
pop1 = 1;           % Complementary mode
ITB = 0;            % Master time base
MDCS = 1;           % Independent duty cycles
CAM = 1;            % Center-aligned
IUE = 0;            % Synchronized updates
DBL_UPDATE = 0;     % Single update per cycle

% Set period for 20 kHz PWM:
maxP1 = 1/20000;    % 50 µs

% Dead-time:
DTH = 1e-6;         % 1 µs high-side dead-time
DTL = 1e-6;         % 1 µs low-side dead-time

% Initial duty cycles (50%):
InitDC = {50, 50, 50};

% Block inputs: Connect FOC algorithm duty cycle outputs

Example 2: Interleaved Boost Converter (Independent Time Base)

% Configuration:
pop1 = 4;           % Independent mode
ITB = 1;            % Independent time base per PWM
CAM = 0;            % Edge-aligned
IUE = 0;            % Synchronized updates

% Different periods for interleaving:
maxP1 = {1/100e3, 1/100e3};  % Both 100 kHz

% Phase shift via InitP2:
InitP2 = {1/100e3, 1/100e3};
% Implement phase shift in PHASE register via TLC customization

% Initial duty cycles:
InitDC = {30, 30};  % 30% each

Example 3: Asymmetric Center-Aligned (Double Update)

% Configuration:
pop1 = 1;           % Complementary
ITB = 1;            % Independent time base
MDCS = 1;           % Independent duty cycles
CAM = 1;            % Center-aligned
DBL_UPDATE = 1;     % Enable double update

% Duty cycle with two values:
InitDC = {[40 60], [45 55], [50 50]};
% [Up-count duty, Down-count duty]

% Use case: Harmonic reduction, EMI optimization

Example 4: ADC Trigger Synchronization

% Trigger ADC at 25% and 75% of PWM period:
TrigOut = [1 0.25*maxP1; 1 0.75*maxP1];
% [PWM_number Trigger_time_in_seconds]

% Connect to ADC block:
% ADC trigger source = "PWM Trigger 1" and "PWM Trigger 2"
% Provides dual sampling per PWM cycle

PWM-ADC Synchronization

Synchronizing ADC sampling with PWM switching is essential for accurate current measurement in motor control and power conversion. The PWM High-Speed module provides trigger outputs to coordinate ADC conversions at optimal sampling points.

Trigger Options for Control Loop Synchronization

Option 1: Timer-Based Trigger (Independent)

Use Case: Simple periodic control where ADC and PWM timing don't need strict coordination.

Option 2: Timer → ADC → Task

Advantage: Task executes immediately when fresh sensor data is available.

Option 3: PWM → ADC → Task (Motor Control Standard)

Configuration using PWM Trigger Outputs:

% In PWM HighSpeed block:
TrigOut = [1 0];  % PWM1, trigger at valley (center-aligned)
% Or for edge-aligned:
TrigOut = [1 maxP1/2];  % PWM1, trigger at mid-period

% In ADC block:
Trigger_Source = 'PWM Trigger 1';
Sample_Time = 'Inherited';  % ADC rate follows PWM trigger

Benefits:

  • Synchronized current sensing with PWM switching
  • Sample when low-side FET is ON (minimal switching noise)
  • Control loop rate automatically matches PWM frequency

Option 4: PWM → Multiple ADC Triggers → Task (Advanced)

Multi-Phase Sampling Configuration:

% Configure multiple trigger points:
TrigOut = [1 0;              % Trigger 1: PWM valley
           1 maxP1/4;        % Trigger 2: 25% into period
           1 3*maxP1/4];     % Trigger 3: 75% into period

% Each trigger can start a different ADC conversion
% Example: Sample Phase A, B, C currents at different PWM timings

Trigger Configuration Best Practices

PWM Mode Recommended Trigger Timing TrigOut Configuration
Center-Aligned (CAM=1) At valley (maximum current flow) TrigOut = [PWM# 0];
Edge-Aligned (CAM=0) Mid-period (stable PWM state) TrigOut = [PWM# maxP1/2];
Interleaved Phases Staggered triggers per phase TrigOut = [1 t1; 2 t2; 3 t3];

Practical Implementation Example

% Motor Control FOC - 3-Phase Current Sensing
% PWM Configuration:
pop1 = 1;              % Complementary mode
ITB = 0;               % Master time base
CAM = 1;               % Center-aligned
maxP1 = 1/20e3;        % 20 kHz PWM

% Trigger at PWM valley for all three phase currents:
TrigOut = [1 0];       % Single trigger at valley

% ADC Configuration:
% - ADC channels AN0, AN1, AN2 for Phase A, B, C
% - Trigger source: PWM Trigger 1
% - Parallel sampling enabled
% - All three conversions complete before task executes

% Simulink Solver Settings:
% Fixed-step size: 1/20e3 (50 µs - matches PWM period)
% Tasking mode: Multitasking

Dynamic Trigger Control

For advanced applications, enable dynamic trigger adjustment:

% Enable dynamic trigger inputs:
TrigOutBlkIn = 1;

% Block will create input ports for each trigger
% Update trigger timing during runtime based on:
% - Operating speed (adjust for different PWM frequencies)
% - Load conditions (optimize sampling point)
% - Fault detection (change timing during recovery)

Troubleshooting

Auxiliary Clock Not Available

Warning: "Auxiliary clock (APLL) not available for this device"

Cause: Device does not support APLL (DOS_00958_pwm_TLCD variant).

Solution: Use standard FCY clock. Reduce switching frequency if resolution insufficient.

Independent Time Base Limitation

Note: When ITB = 0 (master time base), all PWM channels share the same period. Individual period inputs are ignored.

Double Update Mode Requirements

DBL_UPDATE requires:

  • ITB = 1 (independent time base)
  • CAM = 1 (center-aligned mode)
  • MDCS = 1 (independent duty cycles)

If conditions not met, double update is automatically disabled.

Performance Considerations

Resolution vs. Frequency Trade-off

% PWM resolution (bits):
Resolution = log2(FOSC / (Prescaler * f_PWM));

% Example: FOSC = 960 MHz, f_PWM = 100 kHz, Prescaler = 1:
% Resolution = log2(960e6 / 100e3) ≈ 13.2 bits

% Increase FOSC (use APLL) for higher resolution at same frequency

Immediate vs. Synchronized Update

  • IUE = 0 (Synchronized): Updates at period boundary. Eliminates pulse width glitches. Recommended for motor control.
  • IUE = 1 (Immediate): Updates immediately. Use for fast transient response, audio applications, or manual override.
