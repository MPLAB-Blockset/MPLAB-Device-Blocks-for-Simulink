MCHP_TimerInfo - Timer Resource Allocation Display

TimerInfo Block Icon

Description

The MCHP_TimerInfo block visualizes timer resource allocation across all peripheral blocks, showing which timers are used, their configuration, and detecting conflicts.

Display Information

InformationDescription
Timer AllocationWhich peripheral uses which timer
Timer ConfigurationPrescaler, period, mode
ConflictsMultiple peripherals requesting same timer
Available TimersUnused timer resources

Example Display

Timer Resources (dsPIC33AK128MC106):

Timer1: ⚠️ CONFLICT
  - MCHP_Interrupt (Hardware Timer mode)
  - MCHP_IC1 (Input Capture)
  → Resolve: Use different timer

Timer2: PWM_HS Module
  - PWM timebase, Period=1000, Prescaler=1

Timer3: QEI1 Position Measurement
  - Speed calculation, Period=65535

Timer4: [Available]
Timer5: [Available]

Usage

  1. Add MCHP_TimerInfo block to model
  2. Double-click to view allocation
  3. Verify no conflicts exist
  4. Identify available timers for new features
Timer Conflicts: Must be resolved before code generation. Reconfigure peripheral blocks to use different timers.
